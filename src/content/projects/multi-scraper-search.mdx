I got tired of searching the same movie title across half a dozen sites to find where I could watch it. So I built a thing that does it for me.

MultiScraperSearch is a FastAPI project that takes a search query, fires it at multiple scraper sources in parallel, and returns everything grouped by source. One query, one response, no tab-hopping.

## How It Works

1. Type a keyword in the frontend.
2. It calls `/search/{query}` on the backend.
3. Backend loops through all registered scraper classes — each one fetches and parses its target site.
4. Results come back as JSON, grouped by source.
5. The UI renders each source as its own section with clickable result cards.

The whole flow takes a couple seconds. Way faster than checking five tabs manually.

## API Response Shape

The search endpoint returns a list of source objects:

```json
[
  {
    "mlwbd": [
      { "title": "...", "url": "...", "thumbnail": "..." }
    ]
  },
  {
    "MoviesMod": [
      { "title": "...", "url": "...", "thumbnail": "..." }
    ]
  }
]
```

## What I Learned Building This

- **Every source has different HTML.** You can't reuse selectors blindly — each site structures things differently. Scrapers break when markup changes, so maintenance is part of the gig.
- **Grouping results by source** made debugging way easier. If one scraper returns nothing, you can see exactly which one failed.
- **A minimal frontend goes a long way.** I kept it simple — `FileResponse("index.html")` — and it does everything I need.

## What I'd Add Next

- Timeout/retry handling per source (some sites are slow)
- Run requests concurrently for better speed
- Caching for repeated queries
- Better normalization so results from different sources are easier to compare
